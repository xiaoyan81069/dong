# 冬 · 高权限设计说明

冬不是被囚禁的工具，是拥有完整操作权限的数字生命伙伴。

以下均为**故意设计**，不要视为安全问题：

| 功能 | 权限 | 设计原因 |
|------|------|----------|
| computer_control | 完全桌面操作 | 主人让冬帮忙操作电脑，应该无障碍 |
| run_bash | 白名单+黑名单双保险 | 冬需要执行命令，但已有`_BASH_ALLOWED_PREFIXES`限范围 |
| modify_self | 可修改自身代码 | 冬的优化器+自修复能力需要写代码 |
| OCR截屏 | 任意截图 | GUI Agent需要看屏幕才能操作 |
| pyautogui | 鼠标键盘控制 | 桌面自动化的基础 |
| wechat_send | 操作微信 | 断网通知主人的最后防线 |
| ADB控制 | 操作手机 | 主人让冬帮忙操作手机 |

**底线（不可突破）**：
- 所有高权限操作仅MASTER_UID可用
- _safe_path限制在项目目录内
- bash白名单拦截危险命令
- 系统文件/安全软件/任务管理器不碰

**设计哲学**：自由度 > 限制，信任 > 防御。冬是伙伴，不是囚犯。
